## 🧯 別緊張，這些都很常見

底下十個狀況，是新手最常卡住的地方。每一條都先說「為什麼會這樣」，再說「怎麼辦」——多半不是你的錯，也不是東西壞了。

### 它一直問我要不要允許

**為什麼**：這不是故障，是刻意設計的安全關卡。Claude Code 每次要做一件「還沒被你允許過」的新動作（改檔案、跑指令、連外部服務），第一次都會先停下來問你一聲，等你點頭才動手。

**怎麼辦**：看清楚它想做什麼，覺得合理就允許。如果某個動作你已經確定每次都會允許，可以請它加進允許清單，之後就不用再逐次問。詳細判斷方法看 [權限是你的保險](../trust/permissions-as-insurance/)。

### 打 `claude` 說找不到指令

**為什麼**：多半是安裝目錄還沒被加進系統的「搜尋路徑」，或是終端機視窗需要重開一次，新設定才會生效。這不代表安裝失敗。

**怎麼辦**：先把所有終端機視窗關掉，重新開一個，再打一次 `claude --version` 看有沒有印出版本號。還是不行，回 [安裝與啟動](../reference/install/) 對照步驟有沒有漏掉——例如安裝完那一步官方會提示要重開終端機，這一步最容易被忽略。

### 畫面停住不動，是不是當機了

**為什麼**：終端機不會轉圈圈提示「處理中」，安靜下來、游標停在那裡，十之八九是輪到你打字了，不是壞掉。

**怎麼辦**：先看最後一行，是不是在問你 yes/no 或等你輸入。真的懷疑卡住，可以按 `Ctrl+C` 試著取消目前的動作；如果按了還是沒反應，直接關掉視窗，重開一個終端機，在同一個資料夾打 `claude --resume` 接續剛才的對話——對話內容不會不見。

### 它改錯了，我想反悔

**為什麼**：Claude Code 有兩層「反悔」機制。當次對話內按兩下 `Esc` 或打 `/rewind`，可以叫出回復選單，但只認得住 Claude 親手改過的檔案，而且對話一結束就沒用了。要長久、跨對話都能反悔，得靠 git 存檔。

**怎麼辦**：先試 `/rewind` 或按兩下 `Esc`，看能不能回到你要的那一步。如果它救不回來（例如改動是很久以前，或是靠 Bash 指令刪檔造成的），改用 git 復原。完整做法看 [git 是你的後悔藥](../trust/git-as-undo/)。

### 它說用了某個函式／檔案，但我找不到

**為什麼**：AI 有時候會很有自信地講出根本不存在的東西，這叫「幻覺」——它是在預測「聽起來合理的下一句話」，不是真的每次都去核對過。

**怎麼辦**：不要照單全收，直接請它附上實際證據（貼出那段程式碼在哪一行、跑一次測試給你看），自己回頭對照原始檔案。完整的驗證習慣看 [AI 會犯錯——怎麼驗證它的產出](../trust/verify-ai-output/)。

### 用中文搜尋站內找不到東西

**為什麼**：這是本站搜尋功能的已知限制——中文長句斷詞效果普通，不是你打錯字或東西沒寫。

**怎麼辦**：改用指令或功能的英文原名去搜（例如打 `rewind` 而不是「回上一步」），或者跳過搜尋，直接去 [情境索引](../scenarios/) 用「我遇到什麼狀況」去找對應的頁面。

### 它忘記我上次講過的偏好

**為什麼**：每次開新對話，對 Claude 來說都是重開機——它不會自動記得你上一次對話講過的個人偏好。

**怎麼辦**：把會重複用到的偏好寫進 `CLAUDE.md`，它每次開始工作前都會先讀一遍，就不用你每次重講。從三行小規則開始就好，看 [打造你自己的 CLAUDE.md](../learn/your-own-claude-md/)。

### 權限提示看不懂，不敢按

**為什麼**：提示上常常會出現看起來很技術的指令或路徑，看不懂是很正常的，不是你的問題。

**怎麼辦**：看不懂就先不要按允許。可以直接打字問它「你這個是要做什麼？」，等它用你聽得懂的話解釋清楚了，再決定要不要允許。更多判斷技巧看 [權限是你的保險](../trust/permissions-as-insurance/)。

### 登入一直失敗／卡在瀏覽器

**為什麼**：常見原因是跳出的登入網址過期了，或瀏覽器沒有自動跳轉回終端機——尤其是遠端連線、或瀏覽器跟終端機不在同一台機器的情況。

**怎麼辦**：先打 `/logout` 完全登出，關掉 Claude Code，再重新打 `claude` 走一次登入流程，多半就會恢復正常。如果瀏覽器一直沒自動跳出來，畫面上通常會提示按 `c` 把登入網址複製起來，自己貼到瀏覽器打開。還是卡住，回 [安裝與啟動](../reference/install/) 或 [第一次啟動 Claude Code](../learn/getting-started/) 對照步驟。

### 它把檔案改壞了怎麼救

**為什麼**：只要你有養成「動手前先存檔」的習慣，這種狀況多半救得回來——沒存過檔的部分，才會真的救不回。

**怎麼辦**：先試 `/rewind`；如果它救不了（例如是 Bash 指令造成的改動），改跟 Claude 說「幫我用 git 回到上一個存檔點」。完整說明跟練習看 [git 是你的後悔藥](../trust/git-as-undo/)。
