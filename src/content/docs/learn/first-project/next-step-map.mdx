## 🤝 先講清楚：這裡保證什麼、不保證什麼

**保證**：走完 A 或 B 軌道，你會做完一個看得到成果的範本專案，並且在過程中親手撞過、解決過幾個 Claude Code 的核心功能情境。

**不保證**：你自己接下來要做的真正專案一定會順利完成。

每個人的專案內容、環境、卡關的地方都不一樣，沒有人能替你保證你的專案會怎麼樣——包括這個網站。

## 🔁 真正可以帶走的東西：一個可以重複用的循環

功能名稱會變，卡關的細節會變。但下面這個順序不會變：

```text
遇到問題（它卡住了／它又忘了／我想重複用這段）
        ↓
怎麼問（說目標不說步驟，把卡關講清楚）
        ↓
發現功能（權限模式／CLAUDE.md／skill／MCP 之類的機制被自然帶出來）
        ↓
沉澱規則（寫進 CLAUDE.md，或至少記在腦子裡下次會用到）
        ↓
帶著規則回到「遇到問題」，下一次卡關通常更少、更快解決
```

這是你在 A、B 軌道裡實際走過四、五次的循環（回頭看看每一站結尾的「這一站你沉澱了哪條規則」，就是這個循環的證據）。

不管你接下來做的是什麼專案，這個循環都可以直接搬過去用。

## 🗺️ 接下來可以深入的方向

- 想系統化一點學：[Anthropic Academy](../../anthropic-academy/) 有更完整的課程可以跟。
- 想要「規定動作一定要執行」而不只是「Claude 盡量照做」：進階功能 [hooks](../../../reference/hooks/)。
- 想自己動手寫 skill、接自己的 MCP server、寫外掛：這些都屬於開發者軌道。本站的非工程師軌道到「安全使用現成連接器」為止；自建這件事需要寫程式的能力，有興趣可以從 [skills](../../../reference/skills/) 跟 [MCP 連接器](../../../reference/mcp/) 的進階段落開始看。

## ⚠️ 限制與注意

- 誠實講：走過一次範本專案，不代表你以後每次都不會卡關——你會，而且很正常。卡關本身就是這個循環要處理的東西，不是失敗訊號。
- 這個網站不會替你做你的真實專案，能做的是讓你在遇到卡關時，知道怎麼問、去哪裡查。
- 「進度僅存於這台裝置」這件事也適用於你腦中記住的規則——真的常用的偏好，記得寫進 `CLAUDE.md`，不要只放在記憶裡。
