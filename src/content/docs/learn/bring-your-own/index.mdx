練習包做完了，成果也看到了。

接下來很自然會想：那可以拿來處理我自己的東西了吧？但資料夾一打開，又會猶豫——不知道從哪裡開始，也怕不小心把東西弄壞。

這種猶豫很正常。這一課要幫你做兩件事：挑出一個適合當「第一個」的真實專案，然後把開工前該做的安全準備一次做完。

## 🎯 什麼樣的專案適合當第一個

不是隨便一個念頭都適合馬上開工。用這三個判準篩一下：

<CardGrid>
	<Card title="範圍收在一個資料夾內" icon="open-book">
		Claude Code 是以資料夾為工作範圍在做事的。範圍越集中，它越清楚該碰哪些檔案、不該碰哪些，你也越好確認結果。
	</Card>
	<Card title="幾天內看得到成果" icon="approve-check">
		第一個專案是拿來建立信心跟方法的，不是拿來許願的。選一個做完會有明確成果、不會拖上好幾週的事。
	</Card>
	<Card title="做壞了能復原" icon="warning">
		有備份、或本來就可以重來的東西最安全。這樣你才能放心讓 Claude 動手試，不用怕一失手就回不去。
	</Card>
</CardGrid>

## ✅ 開工前三個檢查

挑好專案，別急著馬上讓它碰你的真實檔案。先花幾分鐘做完這三件事：

<Steps>

1. **先複製一份專案資料夾**，或請 Claude 幫你建一次 git 存檔——這是你的復原點。存檔怎麼做，看[「git 是你的後悔藥」](../../trust/git-as-undo/)。

2. **過一遍「什麼不該給 AI」**——資料夾裡有沒有證件照、密碼、別人的個資混在裡面。先讀[什麼不該給 AI](../../trust/what-not-to-share/)，確認過再開工。

3. **在資料夾裡寫下前三行 CLAUDE.md**——不用多，先講清楚你這次想要的結果跟邊界就好。忘記怎麼寫，回去看[打造你自己的 CLAUDE.md](../your-own-claude-md/)。

</Steps>

## 🤝 保證與不保證

我們保證：這套挑選判準、加上這三個開工前檢查，能幫你安全地踏出第一步。

我們**不**保證你的專案一定會順利做完——真實專案比練習包複雜，卡關是正常的，不是你哪裡做錯了。

下一頁教你的，就是怎麼在卡關的時候，跟 Claude 一起把它解開，而不是卡住就放棄。

## ✍️ 跟著做

先用一句話寫下你的專案：「做完這件事，我會得到＿＿」。

接著跑完上面三個開工前檢查。

**預期結果**：你會有一個備份好、過濾過敏感資料、資料夾裡放了三行 CLAUDE.md 的專案資料夾——準備好交給 Claude 動手的狀態。

<details>
<summary>做壞了怎麼辦</summary>

如果檢查跑到一半發現資料夾裡真的有密碼、證件照這類東西：先把它們移出這個資料夾（或整個複製一份乾淨版本再開工），不要留在會被讀進 context 的地方。

如果沒做 git 存檔就已經讓 Claude 動手改了檔案：先停下來，看你複製的那份原始資料夾還在不在——這就是為什麼第一步要先複製一份。之後每次開工前都養成先存檔的習慣。

真的不確定某個東西該不該給？保守一點，先不要給，讀完[什麼不該給 AI](../../trust/what-not-to-share/)再決定。

</details>

## 📝 該進你的 CLAUDE.md 嗎？

跑完三個檢查之後，回頭想想：這次挑專案、做安全檢查的過程中，有沒有哪個習慣，值得寫成一條 CLAUDE.md 規則？
