## 這是什麼

跟 Claude Code 對話，就像交代一個新同事任務：你用白話講你要什麼，它會先看看專案裡的檔案、告訴你打算怎麼做，動手改檔案之前預設會先給你看「打算怎麼改」再問你要不要真的做，改完還能幫你跑測試、寫 git commit。你不用先學一套特殊語法，正常打字描述目標就好。

## 進一步了解

### 開始對話

安裝好之後（見 [安裝與啟動](../install/)），在專案資料夾打 `claude` 就會進入對話畫面。它會先讀取專案結構，不用你手動附加檔案，直接問：

```text
這個專案用了什麼技術？
主要進入點在哪裡？
```

要它動手改東西，一樣用白話描述，例如：

```text
幫主檔案加一個 hello world 函式
```

它會找到合適的檔案、顯示打算怎麼改（diff），等你同意才真的寫入。這個「先問過你」的行為是預設安全機制，不是每次都要你手動關掉。

### 基本操作

| 你想做的事 | 怎麼做 |
| --- | --- |
| 看有哪些指令 | 輸入 `/help` |
| 清空目前對話重新開始 | 輸入 `/clear` |
| 中斷它正在做的事 | 按 `Esc` |
| 離開 Claude Code | 輸入 `/exit` 或按 `Ctrl+D` |
| 直接跑一個 shell 指令、不透過 Claude 判斷 | 開頭打 `!`，例如 `!npm test` |
| 切換權限模式（要不要每次都問你） | 按 `Shift+Tab` 循環切換 |
| 多行輸入 | 行尾打 `\` 再按 Enter |

`claude "任務描述"` 也可以直接帶著任務啟動，不用先進對話再打字；`claude -p "問題"` 則是問完立刻結束，不留在對話裡，適合一次性查詢。

### 常見流程

Claude Code 也擅長 git 操作，可以直接用白話問：

```text
我改了哪些檔案？
幫我用有意義的訊息 commit
```

複雜任務建議拆成步驟描述，而不是丟一句模糊的話：

```text
1. 建一個使用者資料表
2. 建 API 端點讀寫這張表
3. 做一個能看跟編輯資料的網頁
```

## 限制與注意

- **中斷不等於復原**：按 `Esc` 只是停止「目前正在做」的動作，已經完成的檔案修改不會自動撤銷。想回到之前的狀態，可以連按兩次 `Esc`（輸入框為空時）打開回溯選單，或自己用 git 處理；沒有配套的版本控管習慣時，貿然中斷後續清理會比較麻煩。
- **`!` 直接跑指令會跳過確認**：用 `!` 開頭的指令不經過 Claude 判斷，直接在你的機器上執行，等於你自己手動下指令——打之前要知道自己在做什麼，這不是安全網。
- **權限模式切換有風險**：`Shift+Tab` 切到「自動接受編輯」或更寬鬆的模式後，Claude 不會每次改檔案都問你，適合你已經很信任它在做的任務、不適合初次接觸就切換。
- **它不會讀心**：指令越模糊，越容易做出你沒預期的結果；具體描述「哪個檔案」「什麼症狀」通常比一句話「修好那個 bug」更快得到對的結果。
- 對話歷史存在本機、依專案資料夾區分，`/clear` 只是開新的一輪，之前的對話仍可用 `/resume` 找回，不是永久刪除。
