## 🔄 這批更新有沒有感

v2.1.208 到 v2.1.210 這三版，大部分條目是背景任務（`claude agents`）、SDK、Bedrock 等邊角案例的穩定性修正，日常互動使用感受不大。但有三件事值得留意：一是**權限規則寫法**多了一則新警告，二是 **skill／指令參數**不會再被靜默吃掉，三是 **subagent 的 worktree 隔離**修了一個會讓隔離失效的資安 bug。有自己寫過 `settings.json` 權限規則或 skill 的人，建議看完這頁。

## 📋 逐版解讀

### v2.1.208

**日常使用有感**

- 新增**螢幕閱讀器模式**：執行 `claude --ax-screen-reader`、設定環境變數 `CLAUDE_AX_SCREEN_READER=1`，或在 `settings.json` 加 `"axScreenReader": true`，畫面會改用純文字方式呈現，給螢幕閱讀器使用者用。
- 新增 `vimInsertModeRemaps` 設定：vim 模式下可以把兩鍵序列（例如連按兩下 `j`）對應成 Escape，不用去按實體 Esc 鍵。
- 一大批**背景任務（background agents）穩定性修正**：斷線後無法重連、context window 用量顯示錯誤地跳回 100%、記憶體洩漏等。如果你有在用 `claude agents` 讓任務在背景跑，這批修正會讓體驗明顯更穩，但操作方式沒有變。
- `/release-notes` 點「顯示全部」以前會把整份英文 changelog 塞進對話 context，這版修正後不會了——省 token，不影響你怎麼用這個指令。

**細節修正（可略過）**

- 大量 SDK／headless／Bedrock／Windows 平台的邊角案例修正（stream-json 卡死、HTTP/2 斷線、AWS SSO 憑證等），一般互動式使用不會碰到。
- 效能面：多處記憶體洩漏修正、大型 session 恢復速度優化、高工具數場景下每輪處理加速。

### v2.1.209

單一 regression 修正：`/model` 等對話框在背景 session（`claude agents`）裡被上一版的防護規則誤擋住，這版修回來。純內部修正，沒有新行為要學，可略過。

### v2.1.210

**日常使用有感**

- ⚠️ **權限規則新警告**：如果你在 `settings.json` 的 `permissions` 裡寫過 `Write(path)`、`NotebookEdit(path)` 或 `Glob(path)` 這種寫法，啟動時會跳出警告——這幾個工具的權限比對方式跟 `Edit`／`Read` 不同，官方建議改寫成 `Edit(path)` 或 `Read(path)`。本站範例目前沒有用到這幾種寫法，但如果你自己寫過權限規則，值得檢查一次。
- **skill／指令參數不會再被靜默吃掉**：以前呼叫 skill 或 slash command 時，如果帶的參數比 `$1`、`$2` 這類位置變數多或對不上，Claude Code 會直接把多的部分刪掉；這版修正後會原樣保留，不會憑空消失。
- **subagent 的 worktree 隔離修了一個資安相關 bug**：設定 `isolation: 'worktree'` 的 subagent，原本可能繞過隔離、直接對主 repo 跑會修改內容的 git 指令；修正後才真正符合「subagent 在自己的空間做事，不動到你的主目錄」這個設計初衷。同一版也加強了 Agent tool 對間接提示注入的防範（subagent 讀到的檔案內容裡如果藏了指令，不會被誤當成真正的指示執行）。
- 無人值守（unattended）session 的一個卡住成因被修掉：hook 逾時之前會被誤判成「使用者拒絕」，導致排程或背景 session 停住等一個不會來的回覆；這版修正後逾時會正確回報成逾時。

**細節修正（可略過）**

- 一長串背景任務／agent 面板 UI 細節修正（游標行為、圖示殘留、任務進度計數、暫存畫面殘留等）。
- auto mode 的權限分類器行為調整：外部（第三方整合）session 現在預設用 Sonnet 5 判斷權限。
- `ultracode` 關鍵字誤觸發修正：以前 webhook 或 PR 留言等非人類直接輸入的內容裡如果剛好出現這個字，會被誤判成使用者主動啟用；這版修正後只有人類真的輸入才會觸發。
